It is ok to say "CSS variables" instead of "custom properties"

The article is a comprehensive guide to CSS variables (also known as custom properties), detailing their syntax, usage, and capabilities. It begins by explaining the fundamental syntax for declaring and using CSS variables, highlighting the importance of the double hyphen prefix (--) for variable names and the var() function for accessing their values. The article emphasizes that CSS variables are subject to cascading and inheritance, which allows for dynamic updates based on the element to which they are applied, or inherited from the parent element. The author explains the concept of variable scope, demonstrating how variables defined at the :root level can be used globally, while those defined within specific selectors are limited to those elements and their descendants. It covers how to define fallback values within the var() function to handle cases where a variable is not defined. The article delves into the dynamic nature of CSS variables, showcasing how they can be modified using JavaScript to create interactive and responsive designs. It provides examples of how to read and set CSS variable values using JavaScript's getPropertyValue() and setProperty() methods. It notes that changes made to CSS variables via JavaScript will trigger updates in the browser, reflecting the new values in real-time. The guide touches on using CSS variables within media queries, allowing for adaptive styling based on screen size or other media features. It explores the use of CSS variables in complex calculations with calc(), enabling dynamic adjustments of sizes, positions, and other properties based on variable values. The article also discusses the limitations of CSS variables, particularly that they can only hold property values, not selectors or media queries. It mentions browser compatibility for CSS variables, which is generally good across modern browsers, but advises checking for older browser support. The author also explains the use of CSS variables with the cascade, inheritance and specificity to ensure values are applied correctly. It demonstrates how a variable can be overridden at a more specific level, or inherited from a parent if not defined locally. Finally, the article showcases practical applications of CSS variables, such as theming (light vs. dark mode) and component styling, where variables can be used to customize the appearance of UI elements. It emphasizes the power of CSS variables in creating maintainable, scalable, and dynamic stylesheets.

Key Points:

  • CSS variables are declared with --variable-name: value; and used with var(--variable-name, fallback-value);.
  • They are subject to cascading and inheritance, allowing dynamic updates.
  • Variable scope determines where a variable can be accessed (:root for global, selector-specific for local).
  • Fallback values in var() handle cases where a variable is not defined.
  • CSS variables can be manipulated with JavaScript using getPropertyValue() and setProperty().
  • They can be used within media queries for responsive design.
  • CSS variables are compatible with calc() for dynamic calculations.
  • They can only hold property values, not selectors or media queries.
  • Browser compatibility is generally good across modern browsers.
  • CSS variables can be overridden based on cascade, inheritance, and specificity.
  • CSS variables can be used to create scalable, maintainable and dynamic stylesheets through features like theming.
