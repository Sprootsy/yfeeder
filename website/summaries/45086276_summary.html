Summary

The document provided is a changelog for the Curb Ruby library. It details the various changes, bug fixes, and improvements made to Curb over different versions.

Curb 0.9.11 (2024-04-28) is a minor release that addresses a long-standing SSL-related bug, preventing the library from functioning correctly in some environments. This version disables the built-in cert file used by Curb to avoid conflicts with other applications' OpenSSL configurations.

Curb 0.9.10 (2023-12-15) fixes two issues. The first resolves a crash related to using the :on_failure option with Curl >= 7.80.0, where Curl's error buffer is not null-terminated. The second addresses a build error that occurs with newer versions of Ruby and libcurl.

Curb 0.9.9 (2022-03-10) includes two fixes. One prevents a crash when using HTTP/3 (CURLMOPT_PROTOCOL) due to shared curl easy handles. The other resolves an issue with the :body_str option, where a Ruby string was not being passed correctly, leading to potential crashes.

Curb 0.9.8 (2022-02-22) addresses a bug in the :body_str functionality, which was introduced in version 0.9.7.

Curb 0.9.7 (2022-02-21) introduces several changes and bug fixes. It addresses a regression where Curb::Easy#header_str was not populated when using a block. It fixes an issue where Content-Length was being set incorrectly when using put or post requests. It allows setting the request body via :body_str with correct Content-Length handling. It also fixes a memory leak in test code. Additionally, it includes more robust error checking when setting options on easy handles.

Curb 0.9.6 (2021-12-03) includes a fix to ensure that Content-Length is properly set on PUT requests, even when using the multipart: true option.

Curb 0.9.5 (2021-11-10) adds support for HTTP/3 via the CURLMOPT_PROTOCOL option and fixes an issue related to transferring ownership of the multi handle to easy handles within a block.

Curb 0.9.4 (2021-09-08) resolves a crash caused by accessing an already-freed Curl::Easy object.

Curb 0.9.3 (2021-08-10) primarily addresses a memory leak related to multi-handle operations when used with a block. It also includes a change to avoid calling curl_multi_wakeup on Ruby 2.4 and earlier, which can cause problems.

Curb 0.9.2 (2021-04-13) adds support for passing options to curl_multi_setopt using the :multi option.

Curb 0.9.1 (2021-03-16) includes several changes, including the addition of Curl::Easy#header_str, improvements to the documentation to use https URLs, the removal of Travis CI configuration in favor of Github Actions, and a fix to prevent double-freeing of the easy handle in specific scenarios.
