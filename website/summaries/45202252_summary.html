Dotter: Dotfile manager and templater written in Rust

Summary

Dotter is a dotfile manager that utilizes a TOML configuration file to manage the deployment of files and templates. It allows users to specify source paths, target paths, and templating engines for various files and directories within their dotfile repository. Dotter supports multiple profiles, enabling users to apply different configurations based on the system or environment they are working in. It also includes features for variable substitution, allowing for dynamic configuration based on environment variables or user-defined values.

The core functionality of Dotter revolves around reading a TOML configuration file, typically named `dot.toml`, which defines how files and directories should be handled. Each entry in the configuration specifies a source file or directory within the dotfile repository and a corresponding target location on the system. Dotter can then either copy the source file directly to the target location or process it as a template using a specified templating engine, such as Jinja2 or Handlebars. When templating is used, Dotter substitutes variables within the template with values from the environment or user-defined variables.

Dotter's profile system allows users to define multiple configurations within the same `dot.toml` file. Each profile can specify different source and target paths, as well as different templating settings and variables. This enables users to easily switch between different configurations based on the system they are working on, such as a personal laptop or a work computer. Profiles can be activated using the `--profile` flag when running Dotter commands.

In addition to managing files and directories, Dotter also provides features for managing packages and running shell commands. Users can specify a list of packages to be installed on the system, and Dotter will use the system's package manager to install them. Users can also define shell commands to be executed before or after the deployment of files and templates. This allows for further customization and automation of the dotfile deployment process.

Dotter aims to simplify the management of dotfiles by providing a declarative configuration format and a set of tools for deploying and managing files, templates, packages, and shell commands. Its features for profiles, variable substitution, and templating make it a flexible and powerful tool for managing dotfiles across multiple systems and environments.

Key Points

  • Dotter is a dotfile manager that uses a TOML configuration file (e.g., `dot.toml`).
  • It allows specifying source files/directories and target locations.
  • Supports templating engines like Jinja2 and Handlebars for dynamic configuration.
  • Features profile support for different system configurations.
  • Enables variable substitution using environment variables or user-defined values.
  • Can manage packages by using system's package manager.
  • Allows execution of shell commands before/after deployment.
