Beginner-friendly, unofficial documentation for Helix text editor

Summary

The article is a basic introduction to using the Helix editor. It focuses on core concepts and how to perform essential tasks within the editor. Helix is described as a modal editor inspired by Vim, but with some key differences, most notably its focus on multiple selections and a more accessible design for new users.

The guide starts with the fundamental idea of selection as the primary way to interact with the editor. Instead of verbs (commands) followed by nouns (text objects) like in Vim, Helix emphasizes selecting the text first and then applying actions to that selection. Basic movement keys (hjkl) are introduced, as well as ways to move by word (w/b), beginning/end of line (0/$), and beginning/end of file (gg/G). The article describes how to perform text selections using 'v' for selecting characters linewise and 'V' for selecting whole lines. It provides examples of how to extend or shrink selections.

The article introduces the concept of multiple selections, where the same command can apply to multiple disjoint sections of the document simultaneously. It explains how to split selections based on regular expressions or search results ('S'), add selections based on search results ('alt-s'), and keep or remove selections based on matching patterns.

The guide explains how to perform basic editing operations: deletion ('d'), insertion ('i'), appending ('A'), replacing ('c'), and yanking/pasting ('y'/'p'). It touches on more advanced editing features like indenting and unindenting (>/<) and joining lines ('J').

Searching is covered using '/', with 'n' and 'N' to navigate forward and backward through matches. Replacing text is discussed, noting that Helix uses the 's' command for simple replacements within selections.

The article introduces the built-in Language Server Protocol (LSP) support, explaining that Helix automatically configures LSP for supported languages. It highlights features like code completion, go-to-definition, find-references, and rename-symbol. The article also covers formatting code with the '=' key, using the configured LSP server.

The article briefly describes how to use the command palette (<space>:), which allows executing commands by name. Examples include opening files, writing files, quitting, and toggling settings.

Finally, the guide mentions working with multiple buffers (files), using tabs to manage them, and the `:buffer-next` and `:buffer-previous` commands to switch between buffers. It briefly touches on splits using `:vsplit` and `:hsplit`.

Key Points

  • Helix is a modal editor inspired by Vim, but with a focus on multiple selections and more user-friendly design.
  • The primary way to interact with Helix is by making a selection and then applying actions.
  • Basic movement keys are hjkl, along with word (w/b), line (0/$), and file (gg/G) navigation.
  • Selections are made with 'v' (character-wise) and 'V' (line-wise).
  • Multiple selections allow applying the same command to multiple parts of the document.
  • Basic editing operations: delete (d), insert (i), append (A), change (c), yank (y), paste (p).
  • Searching is done with '/', navigating matches with 'n' and 'N'.
  • Replacing text within selections is done with 's'.
  • Helix has built-in LSP support for code completion, go-to-definition, find-references, and renaming.
  • Code formatting is done with '='.
  • The command palette (<space>:) allows executing commands by name.
  • Multiple buffers are managed with tabs, and commands like `:buffer-next` and `:buffer-previous`.
  • Splits can be done with `:vsplit` and `:hsplit`.
