Zed for Windows: What's Taking So Long?

Zed, a high-performance, collaborative code editor, is making progress on its Windows version. The development team has overcome significant challenges related to Windows-specific issues, particularly with GPU integration and input handling. They initially faced difficulties with GPU processes crashing due to conflicts between the Chromium rendering engine and the way Windows handles GPU resources. After extensive investigation, they implemented a workaround by disabling certain Chromium features, which stabilized the application but introduced performance regressions.

To address the performance issues, the team explored different rendering methods, eventually opting to use Direct3D 11 (D3D11) instead of the default OpenGL. This change required substantial modifications to their rendering pipeline but resulted in significant performance improvements and reduced power consumption. They also encountered challenges with input handling, specifically with IME (Input Method Editor) support for languages like Chinese and Japanese. The standard IME APIs in Windows proved unreliable, leading to the development of a custom solution using the Text Services Framework (TSF) for more robust and predictable IME behavior.

The development team also focused on improving the overall user experience on Windows, including addressing issues with font rendering, window resizing, and handling of DPI scaling across multiple monitors with different resolutions. They are actively working on polishing the UI, fixing bugs, and optimizing performance to ensure a smooth and responsive experience for Windows users. While a public release date is not specified, the blog post conveys that the Windows version of Zed is progressing steadily, with the core technical hurdles largely resolved. The team is now focused on refinement and stabilization in preparation for wider testing and eventual release.

Key Points

  • Zed's Windows version is under development and making good progress.
  • Initial development was hampered by GPU process crashes related to Chromium and Windows GPU resource conflicts.
