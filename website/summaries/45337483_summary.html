A collection of technical things every software developer should know (2017)

This GitHub repository, titled "Every Programmer Should Know," is a curated collection of resources and topics intended to guide software developers in their professional growth. It's not a strict checklist but rather a comprehensive roadmap highlighting areas of knowledge that can significantly benefit a programmer's understanding and capabilities.

The repository is organized into several key sections, each covering a different facet of software development and computer science. These sections aim to provide a broad overview of essential concepts and practical skills.

One core area is Source Control, emphasizing the importance of understanding and using version control systems like Git. This includes branching strategies, merging, conflict resolution, and general best practices for collaborative development. The goal is to ensure developers can effectively manage code changes, collaborate with others, and maintain a project's history.

Another section covers General Computer Science concepts. This delves into fundamental topics such as data structures (arrays, linked lists, trees, graphs, hash tables), algorithms (sorting, searching, graph traversal), and basic computer architecture. A solid grasp of these principles enables programmers to write more efficient and effective code by choosing the right data structures and algorithms for a given problem.

Mathematics is also highlighted, pointing out its relevance to various programming tasks. While not every programmer needs advanced mathematical skills, a foundational understanding of discrete mathematics, linear algebra, and statistics can be invaluable for specific domains like game development, data science, and machine learning. The repository includes resources to help developers brush up on or expand their mathematical knowledge.

Operating Systems knowledge is another critical area. This involves understanding how operating systems manage resources, handle processes, and interact with hardware. Topics covered include process management, memory management, file systems, and concurrency. This knowledge helps developers write applications that are more robust, efficient, and well-behaved within the operating system environment.

Databases are a fundamental part of many software applications, and the repository includes resources for learning about database design, SQL, NoSQL databases, and ORM (Object-Relational Mapping) technologies. Understanding database concepts allows developers to effectively store, retrieve, and manage data, ensuring the integrity and performance of their applications.

The section on Networking focuses on the principles of computer networks and how applications communicate over a network. This includes understanding TCP/IP, HTTP, DNS, and other networking protocols. This knowledge is essential for developing web applications, distributed systems, and any application that relies on network communication.

Security is a crucial aspect of software development, and the repository emphasizes the importance of understanding common security vulnerabilities and how to prevent them. Topics covered include authentication, authorization, encryption, and secure coding practices. A focus on security helps developers build applications that are resilient to attacks and protect user data.

The repository also includes sections on Architecture and Design Patterns. Understanding different architectural styles (e.g., microservices, monolithic) and design patterns (e.g., singleton, factory) helps developers create more maintainable, scalable, and robust software systems. This involves making informed decisions about how to structure code and organize components.

Programming Paradigms are also addressed, encouraging developers to explore different ways of thinking about and structuring their code. This includes understanding object-oriented programming, functional programming, and other paradigms. Exposure to different paradigms broadens a developer's perspective and allows them to choose the most appropriate approach for a given task.

Finally, the repository provides guidance on Testing, emphasizing the importance of writing automated tests to ensure the quality and reliability of software. This includes unit testing, integration testing, and end-to-end testing. A strong testing culture helps developers catch bugs early and prevent regressions.

In summary, the "Every Programmer Should Know" repository is a valuable resource for software developers of all levels. It provides a comprehensive overview of essential concepts and technologies, helping developers expand their knowledge, improve their skills, and become more effective software engineers.

Key Points:

  • Comprehensive collection of resources for software developers.
  • Covers Source Control (Git), General Computer Science (data structures, algorithms), Mathematics, Operating Systems, Databases, Networking, Security, Architecture, Design Patterns, Programming Paradigms, and Testing.
  • Emphasizes fundamental concepts and practical skills for building robust, efficient, and secure software.
  • Highlights the importance of continuous learning and expanding knowledge in various areas of software development.
