C++ Web Server on my custom hobby OS

The article is a blog post detailing the author's experience setting up a web server on a retro 32-bit computer, specifically a MIPS-based system running Debian. The author outlines the process of installing and configuring the Lighttpd web server, addressing challenges related to the system's limited resources and the specific Debian distribution used. The post covers essential steps such as installing necessary packages, configuring the web server, and troubleshooting common issues encountered during the setup. The author also explores different methods for transferring files to the retro system. The goal is to have a functional web server that can serve static HTML content and be accessed from other devices on the network.

The author begins by highlighting the hardware setup, which involves a MIPS-based system with Debian installed. The first challenge is ensuring network connectivity, confirming that the retro machine can communicate with other devices on the network. The author then proceeds to install Lighttpd using the apt package manager.

Once Lighttpd is installed, the configuration process begins. The author emphasizes the importance of understanding the Lighttpd configuration file and making necessary adjustments. This includes setting the server's port and document root, which determines the directory from which the web server will serve files. The author provides specific examples of configuration settings and explains how to modify them to suit the retro system's requirements.

Transferring files to the retro system is another challenge addressed in the post. The author explores different methods such as using `scp` (Secure Copy) and Samba. `scp` is a command-line tool that allows secure file transfer over SSH, while Samba enables file sharing over the network using the SMB/CIFS protocol. The author describes the steps involved in setting up Samba and configuring it to allow file access from other computers on the network.

The author also discusses troubleshooting common issues encountered during the setup process. These include problems with network connectivity, file permissions, and web server configuration. The author provides tips and solutions for resolving these issues, drawing from their own experience and research. The post emphasizes the importance of carefully reviewing error messages and consulting online resources to diagnose and fix problems.

Finally, the author demonstrates how to access the web server from another device on the network. This involves opening a web browser and entering the IP address of the retro system, followed by the port number (if the server is not running on the default port 80). The author shows how to display a simple HTML page served by the web server, confirming that the setup is working correctly.

  • Installing and configuring the Lighttpd web server.

  • Adjusting Lighttpd configuration file, including server port and document root.

  • Troubleshooting common issues such as network connectivity and file permissions.
  • Accessing the web server from another device on the network.
