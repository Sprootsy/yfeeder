Parqeye is a data quality tool designed to prevent data swamps by providing automated data quality checks for data stored in Parquet format. It supports both batch and streaming data pipelines, enabling users to validate data against a predefined schema and custom rules. The tool offers features like schema validation, custom rule-based validation, data type validation, null and missing value checks, uniqueness checks, range checks, and regular expression checks.

The tool can be integrated into existing data pipelines using command-line interface or programmatically via a Python API. It generates reports that highlight data quality issues, facilitating quick identification and resolution. Parqeye is extensible, allowing users to define their own custom validation rules using Python. The documentation provides detailed instructions on installation, configuration, usage, and customization.

Key features and functionalities mentioned in the documentation include:

  • Schema Validation: Checks if the Parquet data conforms to a predefined schema.
  • Custom Rule-Based Validation: Allows users to define custom rules using Python code to validate data based on specific business logic.
  • Data Type Validation: Validates that each column contains data of the expected data type.
  • Null and Missing Value Checks: Identifies columns with a high percentage of null or missing values.
  • Uniqueness Checks: Ensures that values in specified columns are unique.
  • Range Checks: Verifies that numerical values fall within a defined range.
  • Regular Expression Checks: Validates that string values match a given regular expression pattern.

The documentation also covers topics such as setting up Parqeye, configuring validation rules, running Parqeye from the command line or programmatically, and understanding the generated reports.
