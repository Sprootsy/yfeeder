Tailscale Down

Summary

On May 16, 2024, Tailscale experienced an incident that caused packet loss and timeouts for users on several Tailscale versions and platforms. The issue stemmed from a bug in the coordination server (control plane) that caused it to incorrectly compute direct routes between nodes. This resulted in some traffic being misrouted, leading to connectivity problems.

The incident began at 16:23 UTC and was mitigated by 17:13 UTC. During this period, affected users experienced a noticeable degradation in network performance, including packet loss and connection timeouts.

The root cause was traced to a recently deployed change to the coordination server. This change introduced a bug that affected the logic for determining direct routes between nodes. Specifically, the server was failing to properly account for certain network configurations, leading to the miscalculation of routes.

The mitigation involved reverting the problematic change to the coordination server. Once the change was reverted, the coordination server began correctly computing routes again, and connectivity was restored for affected users.

Tailscale is taking several steps to prevent similar incidents in the future. These include improving testing and validation procedures for changes to the coordination server, enhancing monitoring and alerting systems to detect routing anomalies more quickly, and implementing more robust rollback mechanisms.

Key Points

  • Incident: Packet loss and timeouts for Tailscale users.
  • Date: May 16, 2024.
  • Time: Started at 16:23 UTC, mitigated by 17:13 UTC.
  • Cause: Bug in the coordination server (control plane) causing incorrect direct route calculations.
  • Impact: Misrouted traffic, packet loss, and connection timeouts for some users.
  • Mitigation: Reverted the problematic change to the coordination server.
  • Prevention: Improved testing, enhanced monitoring, and robust rollback mechanisms.
