Mysterious Intrigue Around an x86 "Corporate Entity Other Than Intel/AMD"

The article discusses a proposal to establish a central registry for x86 opcode extensions that is not controlled by either AMD or Intel. This initiative aims to address the current situation where AMD and Intel independently manage and document x86 opcode extensions, leading to potential conflicts and inefficiencies in the development of software that utilizes these extensions. The lack of a unified registry can create challenges for compiler developers, operating system vendors, and other software creators who need to ensure compatibility and optimal performance across different x86 processors. The proposal suggests creating a neutral, community-driven registry that would serve as a single source of truth for x86 opcode extensions, promoting greater transparency and collaboration within the x86 ecosystem. This registry would ideally include detailed information about each opcode extension, such as its purpose, functionality, and supported processors. By establishing a central registry, the proposal seeks to streamline the development process, reduce the risk of conflicts, and foster innovation in the x86 architecture.

Key Points:

  • Proposal for a central, neutral registry for x86 opcode extensions.
  • Current system relies on AMD and Intel for opcode extension management.
  • Lack of a unified registry causes problems for compiler developers and OS vendors.
  • A central registry would provide a single source of truth for opcode extensions.
  • Aims to improve transparency, collaboration, and reduce conflicts.
  • Would include details about each opcode extension's purpose and supported processors.
  • Seeks to streamline development and foster innovation in the x86 architecture.
