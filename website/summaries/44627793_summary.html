Stdio(3) change: FILE is now opaque

The article discusses the release of OpenBSD 6.8. It highlights that OpenBSD 6.8 is available on various platforms, including amd64, arm64, i386, macppc, riscv64, and sparc64.

The announcement details improvements to hardware support. It mentions that several new drivers have been added or updated. The vmt(4) driver now supports transmit and receive checksum offload. The re(4) driver supports RTL8125 chipsets. The stge(4) driver has been updated, and support for Intel Ice Lake chipsets has been added to the iwm(4) driver. Support for Intel AX210 has been added to the iwx(4) driver. The urtwn(4) driver now supports Realtek RTL8822BU chipsets. The vendor id for the 82540em has been added to the em(4) driver. A workaround for a problem with the Intel I225-V has been added to the em(4) driver, and the cxgbe(4) driver has been updated.

Regarding virtualization, vmd(8) now supports virtio-net checksum offload, which improves network performance in virtual machines. The virtio(4) and vioblk(4) drivers have also been updated for better functionality and reliability.

The announcement mentions improvements to the network stack. There have been various updates to the routing protocols and network utilities. The kernel now includes the TCP Authentication Option (TCP-AO), which can improve TCP security.

Security enhancements are another key aspect. The OpenBSD team continues to focus on security, and OpenBSD 6.8 includes various security improvements and bug fixes. The detailed list of security improvements are not specified in the article.

The announcement also mentions improvements to the system's utilities and applications. Updates to base system tools and libraries are aimed at enhancing system stability and performance. Specific examples of these updates are not provided.

Finally, the article mentions that the OpenBSD team encourages users to donate to support the project.
