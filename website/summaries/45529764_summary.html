Python 3.14 Is Here. How Fast Is It?

The article is a blog post by Miguel Grinberg discussing the performance improvements in Python 3.14 compared to previous versions, specifically Python 3.13, 3.12, and 3.11. The author uses a suite of benchmarks to measure the execution time of different code snippets across these versions. The focus is on identifying which types of operations have seen the most significant speedups or slowdowns in Python 3.14.

The author begins by acknowledging that while Python 3.14 doesn't introduce major new features, it continues the trend of performance enhancements seen in recent Python releases. The post details the methodology used for benchmarking, including the hardware and software environment (macOS Sonoma, an M1 Pro chip, and specific Python versions). It is important to note that the author tests running the code once and the best of three times. The benchmark suite covers a variety of common Python operations, such as addition, multiplication, division, attribute access, dictionary lookups, function calls, and list comprehensions. The code snippets used in the benchmarks are provided within the article, allowing readers to reproduce the results. Also, the standard deviation is provided in the results.

The core of the article presents the benchmark results in a table format, showing the relative performance of each Python version compared to Python 3.13. This makes it easy to identify operations that are faster or slower in Python 3.14. The author then analyzes these results, highlighting specific areas where Python 3.14 shows notable improvements or regressions. For example, dictionary operations show improvements, attribute access also shows improvement, while certain types of arithmetic operations (especially division) shows regressions.

The article also touches on the potential reasons behind the observed performance changes, linking some improvements to specific optimizations made in the Python interpreter. Miguel also mentions the potential trade-offs involved in these optimizations, where improvements in one area might lead to regressions in another. The author explores a possible explanation for the slowdown in certain divisions, which is tied to how the new version of Python handles particular mathematical operations, perhaps sacrificing speed for precision in some cases. The blog also addresses the impact of optimization work that goes into each new version.

The conclusion summarizes the overall performance profile of Python 3.14 based on the benchmark results. The author notes that while some operations have become faster, others have become slower, indicating that the performance impact of Python 3.14 is workload-dependent. The blog post concludes by advising readers to evaluate the performance of their own code when upgrading to Python 3.14, as the actual impact will vary depending on the specific operations used in their applications.

Key Points:

  • Python 3.14 focuses on performance improvements rather than new features, continuing a trend from recent releases.
  • The author uses a benchmark suite covering various Python operations to compare the performance of Python 3.14, 3.13, 3.12, and 3.11.
  • Dictionary operations and attribute access show improvements in Python 3.14 compared to 3.13.
  • Some arithmetic operations, particularly division, show performance regressions in Python 3.14.
  • The performance impact of Python 3.14 is workload-dependent, with some operations faster and others slower.
  • The author advises users to benchmark their own code to assess the impact of upgrading to Python 3.14.
  • The observed performance changes may be due to specific optimizations and trade-offs made in the Python interpreter.
