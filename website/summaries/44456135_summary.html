Poor Man's Back End-as-a-Service (BaaS), Similar to Firebase/Supabase/Pocketbase

Pennybase is a lightweight, in-memory embedded key-value store written in pure Go. It is designed for scenarios where persistence is not a primary concern, such as caching, session management, or temporary data storage. The entire database resides in memory, offering fast read and write operations. The library is implemented in a single Go file, making it easy to integrate into existing projects with minimal dependencies.

The design emphasizes simplicity and ease of use. It provides basic CRUD operations (Create, Read, Update, Delete) through a straightforward API. Data is stored as byte arrays, providing flexibility in handling various data types. The key-value store utilizes a hash map for efficient lookups and provides basic concurrency control using a mutex to protect against race conditions. It offers a simple and effective solution for in-memory data management without the complexity of more robust database systems.

Pennybase does not offer features such as data persistence, transactions, indexing, or complex query capabilities. It is primarily intended for scenarios where data loss is acceptable or where data can be easily reconstructed from other sources. The focus is on performance and simplicity, making it a suitable choice for applications where a lightweight, in-memory key-value store is sufficient.

Key features include:

  • In-memory storage: All data resides in memory for fast access.
  • Simple API: Easy-to-use CRUD operations.
  • Concurrency-safe: Uses a mutex for basic concurrency control.
  • Lightweight: Implemented in a single Go file with minimal dependencies.
  • Flexible data storage: Stores data as byte arrays.

Pennybase is suitable for:

  • Caching
  • Session management
  • Temporary data storage
  • Scenarios where data loss is acceptable

Limitations include:

  • No data persistence
  • No transactions
  • No indexing
  • No complex query capabilities
