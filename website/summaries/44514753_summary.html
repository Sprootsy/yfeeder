This document describes Anna's Message Catalog Protocol (AMCP), a protocol designed for efficient and scalable message catalog distribution and management. Anna's MCP allows clients to query, retrieve, and subscribe to message catalogs, facilitating the dynamic update of application messages, translations, and other localized content. The protocol aims to minimize bandwidth usage and provide robust mechanisms for handling updates and versioning.

AMCP is designed to be a client-server protocol, where a server manages and serves message catalogs, and clients consume them. The protocol relies on a simple request-response model over TCP/IP connections. Clients initiate requests, and the server responds with the requested data or status information. The core components of AMCP include:

  • Catalog Identification: Each message catalog is uniquely identified by a catalog ID. Clients use these IDs to specify which catalogs they are interested in.
  • Versioning: AMCP uses version numbers to track changes to catalogs. Clients can request specific versions or subscribe to updates when new versions become available.
  • Delta Transfer: To minimize bandwidth, AMCP supports delta transfers, where only the changes between two catalog versions are transmitted. This is particularly useful for frequently updated catalogs.
  • Compression: AMCP supports data compression to further reduce the size of transferred data. Common compression algorithms such as gzip are used.
  • Subscription: Clients can subscribe to catalogs to receive automatic updates whenever a new version is published. This allows applications to stay up-to-date without constantly polling the server.

The protocol defines several key message types:

  • GET_CATALOG: Requests a specific version of a catalog.
  • GET_DELTA: Requests the difference between two versions of a catalog.
  • SUBSCRIBE: Subscribes to a catalog for updates.
  • UNSUBSCRIBE: Cancels a subscription.
  • CATALOG_DATA: Contains the catalog data in response to a GET_CATALOG or as a push notification for a subscription.
  • DELTA_DATA: Contains the delta data in response to a GET_DELTA request.
  • ERROR: Indicates an error condition.

AMCP aims to be efficient and scalable. Delta transfers and compression reduce bandwidth usage, while the subscription mechanism allows clients to receive updates without constant polling, reducing server load. The protocol is designed to be easily implementable in various programming languages and platforms. The use of simple request-response messages over TCP ensures compatibility and ease of integration.

The document also covers security considerations. While AMCP itself does not specify any particular security mechanisms, it recommends using transport-layer security (TLS) to encrypt the communication channel and authenticate the server. Additional security measures, such as client authentication and access control, can be implemented at the application layer.

In summary, Anna's Message Catalog Protocol provides a robust and efficient way to manage and distribute message catalogs. Its key features include versioning, delta transfers, compression, and subscription, all designed to minimize bandwidth usage and reduce server load. The protocol's simplicity and flexibility make it suitable for a wide range of applications that require dynamic message updates and localization.
