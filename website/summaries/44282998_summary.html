Modifying an HDMI dummy plug's EDID using a Raspberry Pi

Summary

The article describes how to modify the EDID (Extended Display Identification Data) of an HDMI dummy plug using a Raspberry Pi. An HDMI dummy plug is a small device that simulates a connected display, useful for situations where a physical monitor isn't available or practical, such as remote desktop access or cryptocurrency mining. The standard EDID data embedded in these plugs may not always provide the desired resolution or refresh rate, which can lead to limitations or issues with the simulated display.

The process involves using a Raspberry Pi to read the EDID data from the dummy plug, modify it to specify the desired resolution and refresh rate, and then reprogram the dummy plug with the modified EDID. The author utilizes the cec-utils package on the Raspberry Pi to read the EDID. Specifically, the read-edid command is used to extract the EDID data from the HDMI port where the dummy plug is connected.

The extracted EDID data is typically in a binary format, which is then converted into a human-readable format using a tool like a hex editor. The author uses a hex editor to examine the EDID data and identify the sections that need modification. Key parameters to adjust include the preferred resolution, supported resolutions, and refresh rates.

Once the EDID data is modified, it needs to be written back to the dummy plug. The article outlines a method of flashing the EDID data using the Raspberry Piâ€™s I2C interface. This involves connecting the Raspberry Pi's GPIO pins (specifically the I2C SDA and SCL pins) to the corresponding pins on the HDMI dummy plug. The author emphasizes the importance of identifying the correct pins on the dummy plug, which may require referencing the device's datasheet or using a multimeter.

To write the EDID data, the author uses I2C tools on the Raspberry Pi. The process involves setting the correct I2C address for the dummy plug and then using I2C write commands to send the modified EDID data to the plug's EEPROM (Electrically Erasable Programmable Read-Only Memory). The article highlights the importance of verifying that the write operation is successful, often by reading back the EDID data after the write to confirm that the changes have been applied correctly.

The article also mentions the potential risks involved in modifying the EDID, such as bricking the dummy plug if the process is interrupted or if incorrect data is written. It recommends backing up the original EDID data before making any modifications to allow for restoring the original configuration if necessary.

Finally, the article concludes by emphasizing the benefits of customizing the EDID of an HDMI dummy plug, which includes achieving specific resolutions and refresh rates that may not be supported by the default EDID, thus enhancing the utility of the dummy plug in various applications.
