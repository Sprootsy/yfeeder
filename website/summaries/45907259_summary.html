The GitHub issue is a feature request for Homebrew to automatically uninstall older versions of installed packages after an upgrade. The user, fxcoudert, suggests that when a package is upgraded (e.g., from version 1.2.3 to 1.2.4), the older version (1.2.3) should be automatically uninstalled to save disk space and reduce potential conflicts. They note that other package managers, such as those used in Gentoo, Debian, and Nix, offer this functionality. The rationale is that keeping older versions around is often unnecessary for most users and can lead to clutter.

Several commenters respond with varying opinions. Some users agree with the request, citing disk space concerns and the desire for a cleaner system. They mention that manually cleaning up old versions is a tedious task. Others disagree, arguing that keeping older versions can be useful for rolling back to a previous state if the new version introduces issues or breaks compatibility with other software. They also point out that disk space is relatively cheap, and the potential benefits of keeping old versions outweigh the cost.

Some users suggest alternative solutions, such as providing a command or a flag that users can optionally use to uninstall old versions.
