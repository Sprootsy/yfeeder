Bringing a decade old bicycle navigator back to life with open source software

Summary

The article details the author's journey of reviving a decade-old Teasi One bicycle navigator using open-source software and some creative tinkering. The Teasi One, an Android-based device, suffered from a dead battery and outdated software, rendering it unusable. The author's primary goal was to update the device to a modern Android version and install up-to-date mapping software, enabling it to function as a useful navigation tool again.

The initial steps involved disassembling the device to replace the dead battery. After replacing the battery, the device could boot into Android 2.2, which was very outdated.

The author explored different methods to update the Android version. They first attempted to use the original manufacturer's update tools but found them unreliable and ineffective. They then tried flashing custom ROMs using tools like Livesuit, but encountered issues with driver compatibility and ROM availability.

Eventually, the author successfully flashed a custom ROM based on Android 4.0.4 (Ice Cream Sandwich) using a combination of Livesuit and a compatible ROM image found on a Russian forum. This significantly improved the device's functionality but introduced new challenges, such as a non-functional touchscreen.

To address the touchscreen issue, the author experimented with different touchscreen drivers and configuration files. They identified the correct touchscreen driver and modified the device's configuration files (specifically the `hwconfig.ini` file) to enable the touchscreen.

With the touchscreen working, the author focused on installing and configuring mapping software. They opted for OsmAnd, an open-source offline map application. However, the limited storage space on the device posed a challenge. The author used `adb` to move the OsmAnd data directory to the external SD card, freeing up space on the internal storage.

As a humorous aside, the author also managed to install and run Doom on the device, demonstrating the extent of their tinkering and the device's newfound capabilities.

Finally, the author configured OsmAnd with offline maps, customized the user interface, and adjusted settings for bicycle navigation. They also installed a custom boot animation and configured the device to automatically start OsmAnd upon booting.

The revived Teasi One now functions as a modern bicycle navigator with up-to-date maps and a responsive user interface. The project demonstrates the potential of open-source software and persistent hacking to breathe new life into outdated devices.
