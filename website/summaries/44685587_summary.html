The sad state of font rendering on Linux (2018)

Summary

The article provides a comprehensive guide to managing fonts in Linux, covering various aspects from installation to configuration and troubleshooting. It begins by explaining different types of fonts, including bitmap, vector, and monospace fonts, and the technologies behind them, such as TrueType and OpenType. It details the common locations where fonts are stored in Linux systems, differentiating between user-specific and system-wide font directories.

The piece offers several methods for installing fonts, including using graphical tools like font managers, command-line tools like `apt` or `yum`, and manual installation by copying font files to the appropriate directories. It also explains how to update the font cache to ensure newly installed fonts are recognized by applications.

Configuration is a key focus, with instructions on setting default fonts for the system and individual applications. The article covers customizing fonts in desktop environments like GNOME and KDE, as well as specific applications such as LibreOffice and the terminal. It also touches on advanced configuration options like font hinting and anti-aliasing to improve readability.

Troubleshooting common font-related issues is addressed, including problems with font rendering, missing fonts, and incorrect font configurations. The article provides solutions such as clearing the font cache, verifying font file integrity, and adjusting font configuration files.

Finally, the article discusses web fonts, explaining how to use them in websites and web applications. It covers different web font formats and how to include them using CSS.

Key Points

  • Font Types: Bitmap, vector, and monospace fonts are the primary types used in Linux, with technologies like TrueType and OpenType being prevalent.
  • Font Storage: Fonts are stored in user-specific directories (e.g., `~/.fonts`) and system-wide directories (e.g., `/usr/share/fonts`).
  • Installation Methods: Fonts can be installed via graphical font managers, command-line tools, or manual copying to font directories.
  • Updating Font Cache: After installation, the font cache must be updated using tools like `fc-cache` to ensure fonts are recognized.
  • Configuration: Default system fonts can be configured through desktop environment settings and application-specific configurations.
  • Font Hinting and Anti-Aliasing: These techniques can be adjusted to improve font rendering quality and readability.
  • Troubleshooting: Common issues include font rendering problems, missing fonts, and incorrect configurations, which can be resolved by clearing the font cache or verifying font files.
  • Web Fonts: Web fonts can be used in websites and web applications by including them via CSS.
