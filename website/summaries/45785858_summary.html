Visopsys: OS maintained by a single developer since 1997

Visopsys is an alternative operating system for computers, developed since 1997 by Andreas Gerlich. It aims to be small, efficient, and easily understandable, focusing on portability and running on modest hardware. Visopsys has a monolithic kernel design, meaning most operating system services run within the kernel for performance. It includes a graphical user interface (GUI) based on its own windowing system rather than relying on X11 or other external systems. Visopsys targets POSIX compliance, which is a standard for operating system interfaces, to enhance compatibility with existing software.

Key Points:

  • Alternative Operating System: Visopsys is presented as a distinct operating system, not a modification or derivative of existing OSes like Linux or BSD.
  • Development History: Development started in 1997, indicating a long-term project with ongoing refinement.
  • Developer: Andreas Gerlich is the primary author and maintainer of Visopsys.
  • Design Goals: The project prioritizes being small, efficient, and understandable, likely targeting embedded systems or older hardware.
  • Portability: One of the main focuses is portability, implying the OS can be adapted to different hardware architectures.
  • Hardware Requirements: The operating system is designed to run on modest hardware, suggesting low resource consumption.
  • Monolithic Kernel: Visopsys uses a monolithic kernel design, which can offer performance advantages but may impact modularity.
  • Graphical User Interface: Visopsys includes its own GUI and windowing system, avoiding reliance on external systems like X11.
  • POSIX Compliance: The OS aims for POSIX compliance to allow easier porting of software and compatibility with standard tools.
