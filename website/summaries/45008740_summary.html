Ghrc.io appears to be malicious

The article is a blog post detailing the author's investigation into unusual network activity originating from a binary named ghrc, located in the /tmp directory of a Linux server. The author first noticed the high CPU usage attributed to this process. Initial checks revealed that the binary was executed from the /tmp directory, which is suspicious because typical system binaries are not located there. Furthermore, the binary seemed to have been created very recently.

Using netstat, the author discovered that ghrc was establishing multiple outgoing TCP connections to seemingly random IP addresses on port 443 (HTTPS). This behavior, combined with the unknown origin and location of the binary, raised serious concerns.

The author then examined the binary using strings to find any readable text within it. The output contained several base64 encoded strings, which, when decoded, revealed URLs pointing to domains with names similar to cryptocurrency exchanges or mining pools. This suggested that the binary could be involved in cryptojacking or other cryptocurrency-related malicious activities.

To further understand the binary's behavior, the author used strace to trace the system calls made by the process. The strace output showed the binary repeatedly calling clone, creating numerous child processes. These child processes were then attempting to connect to the aforementioned IP addresses. This confirmed the earlier suspicion of network-based malicious activity.

The author also noticed the use of futex system calls, which are often used for synchronization in multithreaded applications. This, combined with the cloning behavior, suggested that ghrc was a multithreaded application designed to perform network operations concurrently.

Based on the analysis, the author concluded that the ghrc binary was likely malicious, exhibiting characteristics of cryptojacking malware due to its high CPU usage, suspicious network connections to cryptocurrency-related domains, and the use of threading and cloning to maximize its impact. The author recommended removing the binary and investigating how it was initially placed on the system to prevent future infections.

Key Points:

  • Suspicious binary named ghrc found in /tmp.
  • High CPU usage attributed to the ghrc process.
  • Multiple outgoing TCP connections to random IP addresses on port 443.
  • Base64 encoded strings within the binary revealed cryptocurrency-related URLs.
  • strace output showed repeated clone system calls.
  • Likely a cryptojacking malware.
  • Recommendation to remove the binary and investigate the source of infection.
