Human Fovea Detector

This ShaderToy shader creates a visually interesting looping animation of a cellular automata-like pattern. The core of the effect is based on a 2D grid where each cell's state (color) is determined by the states of its neighbors in the previous frame. The shader doesn't store the grid explicitly in memory. Instead, it implicitly calculates the cell states based on their coordinates and the time.

The animation relies on a combination of modulo operations, sine waves, and fractional parts of numbers to generate the patterns. The code defines several key functions:

  • hash(p): This function takes a 2D vector as input and returns a pseudo-random number. It is used to introduce variation and randomness into the cellular automata.
  • f(p, t): This is the core function that defines the cellular automata rule. It takes a 2D vector (representing a cell's coordinates) and time as input. It calculates a value based on the neighboring cells, using the hash function and trigonometric functions.

The mainImage function is the entry point of the shader. It calculates the color of each pixel on the screen. It does this by:

  1. Calculating the UV coordinates (normalized coordinates) of the pixel.
  2. Scaling and offsetting the UV coordinates to map them to the grid space of the cellular automata.
  3. Calling the f(p, t) function to determine the value of the cell at the calculated coordinates and time.
  4. Mapping the value returned by f(p, t) to a color using a simple color gradient.

The animation is driven by the iTime variable, which represents the elapsed time since the shader started. As time progresses, the f(p, t) function generates different patterns, creating the animation. The use of sine waves and fractional parts of numbers ensures that the animation loops smoothly.

The specific parameters and functions used in the f(p, t) function determine the exact characteristics of the pattern. By modifying these parameters, the author can create a wide variety of different animations.

Key Points:

  • Cellular automata-like animation without explicit grid storage.
  • Uses pseudo-random numbers (hash) and trigonometric functions (sin) to define the cellular automata rule (f).
  • The f function calculates cell states based on neighboring cells and time.
  • The mainImage function maps cell states to colors to create the visual output.
  • The animation loops smoothly due to the use of sine waves and fractional parts of numbers.
  • The iTime variable drives the animation.
