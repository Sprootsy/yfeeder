Forgejo v13.0 Is Available

Forgejo version 1.20.0 has been released with several new features, bug fixes, and notable changes. The release focuses on enhancing user experience, security, and overall system functionality.

A significant update is the introduction of Repository Roles, which allow administrators to grant granular permissions to users within a repository, such as read-only access, triage capabilities, or write access. This enhances security and collaboration by providing more precise control over who can perform specific actions.

The release also includes improvements to the web interface, such as better handling of very long lines, and improvements to the display of issues. These changes aim to improve the overall usability of the platform.

For administrators, the update introduces the ability to specify different SSH port and address bindings, offering greater flexibility in server configuration.

Several bug fixes address issues related to authentication, repository mirroring, and other areas, improving the stability and reliability of Forgejo.

The update also includes dependency updates. These updates improve the security and performance of Forgejo by incorporating the latest improvements from upstream projects.

Users are encouraged to review the complete changelog for a comprehensive list of changes and to follow the upgrade instructions provided in the documentation.

Key Points:

  • Repository Roles: Introduction of granular repository permissions (read, triage, write).
  • UI Improvements: Enhanced handling of very long lines and display of issues.
  • SSH Configuration: Ability to specify different SSH port and address bindings.
  • Bug Fixes: Resolution of various issues related to authentication, repository mirroring, and other areas.
  • Dependency Updates: Updated dependencies for improved security and performance.
