The GitHub issue discusses the addition of a <documentpicture> element to HTML, which aims to address the challenge of providing appropriate image resources for different display contexts, such as varying screen densities and resolutions, print media, and potentially even future display technologies like VR/AR. The current <picture> element is considered insufficient because it is limited to image selection based on viewport size and media queries, but lacks features like content-based selection or finer-grained control over image types and encodings. The proposed <documentpicture> element seeks to provide a more comprehensive solution, enabling developers to specify a range of image resources with associated metadata, allowing the browser to select the most suitable resource based on a wider array of criteria.

One of the primary motivations behind the proposal is to optimize image delivery for various devices and network conditions, improving page load times and reducing bandwidth consumption. By providing multiple image options, the browser can choose the smallest acceptable image for a given device, preventing the unnecessary download of high-resolution images on devices with lower screen densities. The element could also facilitate the selection of different image formats based on browser support and performance characteristics, for example, using WebP or AVIF formats when available.

The discussion includes considerations for the element's attributes, such as how to specify image sources, media queries, and other selection criteria. There's also consideration on how the element would interact with CSS and JavaScript, and how it might be extended in the future to support more advanced features. A key aspect is to ensure that the <documentpicture> element is both flexible enough to meet the needs of developers and efficient for browsers to implement.

One of the challenges involves defining a clear and intuitive syntax for specifying image resources and their associated metadata. Another challenge lies in ensuring that the element is compatible with existing web standards and practices. The discussion explores different approaches to address these challenges, taking into account the trade-offs between complexity and functionality. The end goal is to create a new HTML element that significantly improves image delivery and rendering across a wide range of devices and contexts.

The issue also covers the potential use cases and benefits of the <documentpicture> element. This includes improved performance, reduced bandwidth consumption, and enhanced user experience. The new element will also give more fine grained control to developers in choosing the correct image for a certain device or technology.

Key Points:

  • Proposal: Introduction of a new <documentpicture> element to HTML.
  • Problem: Existing <picture> element is limited in its ability to select appropriate image resources for diverse display contexts.
  • Goal: Provide a more comprehensive solution for image selection based on various criteria like screen density, media type, and browser support.
  • Benefits: Improved page load times, reduced bandwidth consumption, and enhanced user experience through optimized image delivery.
  • Considerations: Syntax, compatibility with existing standards, and future extensibility.
  • Motivation: Optimizing image delivery for different devices and network conditions.
  • Functionality: Allows developers to specify multiple image resources with associated metadata for the browser to choose from.
