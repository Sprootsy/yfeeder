Shipping textures as PNGs is suboptimal

Summary:

The article "Stop Shipping PNGs" by Mason outlines the inefficiencies and drawbacks of using PNG images as the primary image format for web applications and games, especially when alternatives like WebP and AVIF offer significant advantages in terms of file size and compression. The author argues that while PNG has been a reliable and widely supported format, its lossless compression method often results in larger file sizes compared to newer formats that employ more advanced compression algorithms.

The article begins by acknowledging the historical context of PNG's popularity, stemming from its open-source nature and the patent encumbrances associated with GIF. However, it emphasizes that the web development landscape has evolved, and modern browsers and devices now offer native support for superior image formats.

A significant portion of the article is dedicated to comparing PNG with WebP and AVIF. WebP, developed by Google, is highlighted for its ability to provide both lossless and lossy compression, often achieving substantially smaller file sizes than PNG without sacrificing image quality. AVIF, based on the AV1 video codec, is presented as an even more advanced option, offering even better compression efficiency than WebP. The author provides data and examples illustrating the potential savings in file size when switching from PNG to these alternative formats.

The author addresses the concern of browser compatibility by recommending a strategy of using the <picture> element in HTML. This allows developers to provide multiple image formats, enabling browsers to select the most efficient format they support. This ensures that users with modern browsers benefit from the smaller file sizes of WebP or AVIF, while users with older browsers still receive a PNG version of the image, maintaining backwards compatibility.

The article also discusses the impact of image optimization on website performance. Smaller image file sizes translate to faster loading times, reduced bandwidth consumption, and improved user experience. This is particularly important for mobile users and those with slower internet connections.

Furthermore, the article touches on the tooling and resources available to convert PNG images to WebP or AVIF. It mentions various command-line tools and libraries that can automate the conversion process, making it easier for developers to adopt these newer formats.

In conclusion, the article advocates for a shift away from PNG as the default image format for web projects. It urges developers to embrace WebP and AVIF, leveraging the <picture> element to ensure broad compatibility and optimize website performance through reduced image file sizes. The author positions this transition as a practical and impactful step towards a more efficient and user-friendly web.

Key Points:

  • PNG is an outdated image format that results in larger file sizes compared to newer formats like WebP and AVIF.
  • WebP and AVIF offer superior compression algorithms, leading to smaller file sizes without significant loss of image quality.
  • Using the <picture> element in HTML allows developers to serve different image formats based on browser support, ensuring compatibility.
  • Smaller image file sizes improve website loading times, reduce bandwidth consumption, and enhance user experience.
  • Tools and resources are available to easily convert PNG images to WebP and AVIF.
  • Adopting WebP and AVIF is a practical step towards a more efficient and user-friendly web.
